# KGH1012: Prefer consistent multiline arguments

## Summary

When an argument list spans multiple lines, use a consistent layout by
placing either one argument per line or all arguments on a single line.

## Rationale

This rule improves readability by making each argument easier to
identify and compare.

## Before

```csharp
await UpdateSpoolAsync(
    shortCode,
    weight: null, locationShortCode);
```

## After

```csharp
// Option 1: UseOneArgumentPerLine
await UpdateSpoolAsync(
    shortCode,
    weight: null,
    locationShortCode);

// Option 2: UseSingleLineArgumentList
await UpdateSpoolAsync(shortCode, weight: null, locationShortCode);
```

## Applies to

- Method arguments
- Constructor arguments

## Code fixes

- `UseOneArgumentPerLine`: Places one argument on each line.
- `UseSingleLineArgumentList`: Places all arguments on a single line.

## Configuration

None.
