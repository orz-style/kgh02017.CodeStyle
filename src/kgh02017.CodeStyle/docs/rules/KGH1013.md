# KGH1013: Prefer consistent multiline parameters

## Summary

When a parameter list spans multiple lines, use a consistent layout by
placing either one parameter per line or all parameters on a single line.

## Rationale

This rule improves readability by making each parameter easier to
identify and compare.

## Before

```csharp
private async Task<string> UpdateSpoolAsync(
    string shortCode, double? weight, string? locationShortCode)
{
}
```

## After

```csharp
// Option 1: UseOneParameterPerLine
private async Task<string> UpdateSpoolAsync(
    string shortCode,
    double? weight,
    string? locationShortCode)
{
}

// Option 2: UseSingleLineParameterList
private async Task<string> UpdateSpoolAsync(string shortCode, double? weight, string? locationShortCode)
{
}
```

## Applies to

- Method declarations
- Constructor declarations
- Local function declarations

## Code fixes

- `UseOneParameterPerLine`: Places one parameter on each line.
- `UseSingleLineParameterList`: Places all parameters on a single line.

## Configuration

None.
