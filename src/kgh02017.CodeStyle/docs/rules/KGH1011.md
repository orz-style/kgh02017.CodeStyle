# KGH1011: Prefer collection expression

## Summary

When the target collection type can be inferred, use a collection expression.

## Rationale

This rule reduces boilerplate and uses the modern C# collection syntax.

## Before

```csharp
List<string> names = new()
{
    "Alice",
    "Bob",
};
```

## After

```csharp
List<string> names =
[
    "Alice",
    "Bob",
];
```

## Applies to

- Collection initializers

## Code fixes

- `PreferCollectionExpression`: Replaces collection initialization with a collection expression.

## Configuration

None.

## Requirements

- C# 12 or later
