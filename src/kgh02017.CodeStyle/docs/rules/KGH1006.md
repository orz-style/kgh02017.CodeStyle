# KGH1006: Prefer switch expression

## Summary

When using a `switch` statement, use a `switch` expression instead.

## Rationale

This rule reduces boilerplate and improves readability by using the more concise
`switch` expression syntax.

## Before

```csharp
string text;

switch (value)
{
    case 1:
        text = "One";
        break;

    case 2:
        text = "Two";
        break;

    default:
        text = "Other";
}
```

## After

```csharp
string text = value switch
{
    1 => "One",
    2 => "Two",
    _ => "Other",
};
```

## Applies to

- `switch` statements

## Code fixes

- `PreferSwitchExpression`: Replaces a `switch` statement with a `switch` expression.

## Configuration

None.
