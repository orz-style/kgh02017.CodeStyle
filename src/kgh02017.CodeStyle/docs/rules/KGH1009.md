# KGH1009: Prefer ArgumentNullException.ThrowIfNull

## Summary

When validating arguments, use `ArgumentNullException.ThrowIfNull` instead of 
throwing `ArgumentNullException` manually.

## Rationale

This rule reduces boilerplate and promotes the standard .NET pattern for argument validation.

## Before

```csharp
if (value is null)
{
    throw new ArgumentNullException(nameof(value));
}
```

## After

```csharp
ArgumentNullException.ThrowIfNull(value);
```

## Applies to

- Manual `ArgumentNullException` throws


## Code fixes

- `PreferThrowIfNull`: Replaces a manual null check with `ArgumentNullException.ThrowIfNull()`.

## Configuration

None.
