# KGH1007: Prefer assignment line break

## Summary

When the right-hand side of an assignment spans multiple lines, place the assigned
value on the following line.

## Rationale

This rule makes assignments easier to scan by separating the left-hand
side from multiline expressions.

## Before

```csharp
IEnumerable<int> values = Enumerable.Range(1, 10)
    .Where(x => x > 5)
    .ToList();
```

## After

```csharp
IEnumerable<int> values =
    Enumerable.Range(1, 10)
        .Where(x => x > 5)
        .ToList();
```

## Applies to

- Local variable declarations
- Assignment expressions (`=`, `+=`)
- Object initializers


## Code fixes

- `PreferAssignmentLineBreak`: Places the assigned value on the following line.

## Configuration

None.
