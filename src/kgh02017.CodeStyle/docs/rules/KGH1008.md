# KGH1008: Prefer named arguments for literals

## Summary

When passing `true`, `false`, or `null` literals as arguments, use named arguments.

## Rationale

This rule makes the intent of literal arguments explicit and improves readability.

## Before

```csharp
await UpdateSpoolAsync(
    shortCode,
    null,
    locationShortCode);
```

```csharp
SetOption(true);
```

## After

```csharp
await UpdateSpoolAsync(
    shortCode,
    weight: null,
    locationShortCode);
```
```csharp
SetOption(enabled: true);
```

## Applies to

- `bool` literals
- `null` literals

## Code fixes

- `PreferNamedArgumentsForLiterals`: Places argument names for `bool` and `null` literal arguments.

## Configuration

None.
